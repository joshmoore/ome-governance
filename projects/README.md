# **OME Registered Projects (ORP)**

## **1. Overview**

OME Registered Projects (ORPs) are the primary software, standards, and
infrastructure projects that together comprise the OME ecosystem. Each ORP
operates under its own charter and roster while aligning with the shared
principles and governance of OME. This document records the current roster of
Registered Projects as well as the process for modifying this document.

## **2. Current Projects**

| Project      | Scope                                        |
| :------      | :----                                        |
| Core Model   | OME-XML, OME-TIFF, OME Data Model            |
| Bio-Formats  | Format Reader support, Metadata completeness |
| OMERO.server | API stability, Microservices, Integrations   |
| OMERO.web    | Plugin ecosystem, Usability                  |
| IDR          | Reference datasets, Curation workflows       |
| NGFF         | RFC process, Format standardization          |

## **3. Roles and Responsibilities**

Each Registered Project shall:

- Maintain a charter describing its scope and governance.
- Maintain a roster identifying its maintainers and other formal roles.
- Operate in accordance with the OME governance framework.
- Coordinate with other Registered Projects where responsibilities overlap.
- Escalate cross-project or governance matters to the OME Management Group (OMG), where appropriate.

The OMG is responsible for maintaining this roster, approving the creation of
new Registered Projects, and retiring or consolidating projects when necessary.

Individuals or groups interested in proposing a new Registered Project are
encouraged to contact the OMG.

## **4. Document Maintenance**

This roster is maintained by the OMG and updated as needed to reflect the
evolution of the OME ecosystem. Changes to the list of Registered Projects
shall be made in accordance with the decision-making procedures defined by the
OMG Charter.
